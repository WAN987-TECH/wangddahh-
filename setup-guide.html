邻选 · 部署使用指南

按照以下步骤，大约需要 15 分钟 完成全部设置。全程免费，无需购买服务器或域名。

第一步：注册 Supabase 账号

1

Supabase 是一个免费的开源数据库服务，我们用它来存储商品和订单数据。

  1. 用浏览器打开 supabase.com
  2. 点击右上角 "Start your project"
  3. 选择 "Sign in with GitHub"（推荐，最方便）
  4. 如果没有 GitHub 账号，也可以用邮箱注册
免费版提供 500MB 数据库空间和每月 5GB 流量，对于小区团购来说绰绰有余。

第二步：创建数据库项目

2
  1. 登录后点击 "New project"
  2. 填写以下信息：
    • Name：填 groupbuy（或任意名字）
    • Database Password：设置一个数据库密码（建议用浏览器生成的随机密码，记下来备用）
    • Region：选择 Northeast Asia (Tokyo) 或 Southeast Asia (Singapore)，离国内最近，速度最快
  3. 点击 "Create new project"，等待 1-2 分钟让数据库创建完成

第三步：初始化数据库表

3

在 Supabase 项目面板中，左侧菜单找到 SQL Editor，执行以下操作：

  1. 点击 "New query"
  2. 打开 supabase-schema.sql，复制全部内容
  3. 粘贴到 SQL Editor 中
  4. 点击右下角绿色 "Run" 按钮
执行成功后，你会看到 products（商品）、orders（订单）、rounds（轮次）、settings（设置）等表和自动创建的安全策略。这意味着数据库已经准备就绪。

第四步：获取连接信息

4

在 Supabase 项目面板左侧菜单，找到 Settings > API：

  1. 复制 Project URL（格式如 https://xxxxx.supabase.co）
  2. 复制 anon public key（一长串以 eyJ 开头的文字）
  3. 把这两个信息保存到记事本备用，下一步用到
请使用 anon public key（公开密钥），不要使用 service_role key（私密密钥）。service_role key 会绕过所有安全策略，不应该在网页中使用。

第五步：部署网页到 GitHub Pages

5

GitHub Pages 是免费、稳定的静态网页托管服务，无需实名认证。

  1. 打开 github.com，登录你的 GitHub 账号
  2. 点击右上角 "+" → "New repository"
  3. Repository name 填 Community-Group-Purchase-Project（或任意英文名）
  4. 选 Public，勾选 "Add a README file"，点 "Create repository"
  5. 进入仓库，点 "Add file" → "Upload files"
  6. 把以下文件拖进去，点 "Commit changes"：
    • order.html — 核心下单工具（团长 + 顾客）
    • landing.html — 项目宣传页面
    • setup-guide.html — 部署使用指南
    • admin-guide.html — 团长操作手册
    • customer-guide.html — 邻居下单指南
    • supabase-schema.sql — 数据库脚本（备存）
  7. 点仓库顶部 Settings → 左侧 Pages
  8. Source 选 "Deploy from a branch"，Branch 选 main，文件夹选 / (root)，点 Save
  9. 等 1-2 分钟，页面上出现绿色提示框显示网址：https://你的用户名.github.io/仓库名/
你的下单页面地址就是：https://你的用户名.github.io/仓库名/order.html
邻居扫码或点击这个链接就能进入下单页面。

备选方案：Cloudflare Pages

如果 GitHub 访问慢，也可以用 Cloudflare Pages（同样免费）：

  • 打开 dash.cloudflare.com，注册后进入 Workers & Pages → Create → Pages → Upload assets
  • 把所有 HTML 文件拖进去 → Deploy，获得 https://xxx.pages.dev 网址

第六步：首次使用配置

6

用浏览器打开你刚部署的网址，页面会显示「请选择你的小区」。

方式一：如果项目已预设你的小区，直接点击小区名称按钮即可自动连接（无需手动填 URL 和 Key）。

方式二：手动配置，点击「其他小区？手动配置」：

  1. 在 Supabase URL 输入框粘贴第四步复制的 Project URL
  2. 在 Anon Key 输入框粘贴第四步复制的 anon key
  3. 点击 "手动连接"

连接成功后会自动进入顾客下单页面。

然后设置管理员密码：

  1. 在网址后面加上 #admin（如 https://你的用户名.github.io/仓库名/order.html#admin），回车
  2. 这是首次进入，输入你想设置的管理密码和确认密码（密码需 ≥4 位）
  3. 进入后台后，在「轮次设置」标签页可以修改密码
请务必记住你的管理员密码。如果忘记了，需要在 Supabase 的 SQL Editor 中手动清空 admin_password 记录来重置。

第七步：日常使用流程

发起一次团购的完整流程

1. 进入后台：浏览器打开 你的网址#admin，输入管理密码。

2. 创建团购轮次：后台 →「轮次设置」→「创建新轮次」

  • 填写轮次名称（如「5 月第一周团购」）
  • 可选设置下单截止时间（到期后顾客端自动关闭下单）
  • 新轮次创建后状态为「策划中」，在添加商品后点击「开团」才会在顾客端显示
  • 轮次有四种状态：策划中（不可下单）、进行中（顾客可下单）、已截止（到期自动关闭）、已停止（手动停止）
允许多个轮次同时「进行中」，不同轮次的商品和订单完全隔离。

3. 添加商品：后台 →「商品管理」→ 选择轮次 →「添加商品到「轮次名」」

  • 商品名称（必填）— 如「烟台红富士苹果」
  • 商品图片（选填）— 支持 JPG/PNG/GIF/WebP，最大 5MB
  • 规格与价格（至少一个）— 如「500g / 15 元」「1kg / 28 元」
  • 整装拆分计算器：山姆等大包装商品，输入整装数量和总价，设置每份大小，自动计算份数和单价
  • 商品标签（选填）— 如「蛋糕」「冷藏」，用空格分隔
  • 库存数量（选填）— 不填则不限库存，卖完自动显示「已售罄」
商品保存后默认上架，可在列表中点「下架」暂时隐藏。

4. 开团 + 生成二维码：

  • 商品添加完毕后，到「轮次设置」点击该轮次的「开团」按钮
  • 进入「下单二维码」Tab，二维码自动生成
  • 长按二维码图片 → 保存到手机相册 → 发到业主微信群
  • 也可点击「复制下单链接」直接分享

5. 查看订单：后台 →「订单汇总」

  • 顶部下拉选择轮次，按人分组显示订单明细和金额
  • 底部「采购清单」按商品汇总采购数量，支持一键复制
  • 「导出 Excel」下载含订单明细和商品汇总两个 sheet 的表格
  • 对需要取消的订单可点击「砍单」按钮，填写原因后顾客端会显示

6. 收款：后台 →「收款清单」

  • 设置本轮费用（运费、包装费等），系统自动按人数均摊
  • 底部「收款清单」列出每人应付金额（商品金额 + 均摊费用）
  • 「一键复制收款清单」后粘贴到微信群收款

顾客如何使用

  1. 在微信群里长按二维码 → 识别图中二维码
  2. 打开页面后，看到当前在售商品，可搜索、按标签筛选
  3. 点击商品选择规格和数量（支持拆分商品，按份购买）
  4. 选好后点击底部「去下单」
  5. 填写群昵称（必填）+ 备注（选填）
  6. 提交订单，完成
  7. 如需修改，点击页面上方「查/改订单」，输入昵称查询
  8. 订单卡片会显示所属轮次和状态，被团长砍单会显示红色砍单原因

其他实用功能

  • 商品导入导出：「商品管理」→「导出商品」下载 Excel，「导入商品」可批量恢复
  • 多轮次同时运行：允许多个团购轮次同时「进行中」，适合不同品类分开团（如水果团 + 日用品团）
  • 修改站点标题：「轮次设置」→ 修改「顾客端标题」，顾客端顶部会动态更新
  • 修改管理密码：「轮次设置」→ 管理密码区域，输入新密码保存

忘记密码

在 Supabase → SQL Editor 执行：

UPDATE settings SET value = '' WHERE key = 'admin_password';

然后重新打开 #admin 设置新密码。

温馨提示：Supabase 和 GitHub Pages 的免费额度对于一个小区的团购来说完全足够。如果未来你的团购规模扩大，Supabase 的付费方案也只要 $25/月起步。
